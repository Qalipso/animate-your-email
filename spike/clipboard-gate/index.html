Test procedure: click a strategy's button (or manually copy for Strategy 3), then paste into a real Gmail/Outlook compose window. Record the result in the test matrix in DEC-002. Do not judge success from this page — only from what actually appears in the composer / sent / received email.

Strategy 1 — navigator.clipboard.write() (text/html + text/plain)

The mechanism DEC-001 actually proposes to ship. Writes an HTML fragment containing an <img> pointing at the public GIF URL, plus a plain-text fallback.

Strategy 2 — native DOM/selection copy (execCommand)

Selects the real rendered <img> element on this page via a Range and triggers the browser's native copy command, instead of constructing the clipboard payload by hand.


Strategy 3 — manual selection + Cmd/Ctrl+C (control case)

No JavaScript. Manually select the image below (click-drag to select it, or right-click → Copy Image) and press Cmd/Ctrl+C yourself. This establishes the ceiling: if even a human copy/paste doesn't survive the full path, no JS-driven mechanism will either.

Read back what's actually on the clipboard (sanity check only)

This does not tell you whether Gmail/Outlook will render it correctly — only that the write succeeded and what MIME types are present. Some browsers block programmatic clipboard reads outside a user gesture; click right after copying.
